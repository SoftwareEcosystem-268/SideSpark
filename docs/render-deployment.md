# Render Deployment Guide

คู่มือนี้อธิบายการ deploy SideSpark บน Render โดยตรง ทั้งแบบใช้ `render.yaml` และแบบสร้าง service เอง

## Deployment Options on Render

### Option A: Blueprint Deployment

เหมาะเมื่อ:

- ต้องการ deploy ทั้ง frontend, backend, และ database จาก repository เดียว
- ต้องการให้ค่าพื้นฐานมาจาก [`render.yaml`](../render.yaml)

### Option B: Manual Service Setup

เหมาะเมื่อ:

- ต้องการ deploy เฉพาะ backend บน Render
- ต้องการควบคุม service แต่ละตัวด้วยมือ

## Option A: Deploy with `render.yaml`

### What the Blueprint Creates

จาก [`render.yaml`](../render.yaml) จะสร้าง:

- `sidespark-frontend` เป็น Node Web Service
- `sidespark-backend` เป็น Node Web Service
- `sidespark-db` เป็น PostgreSQL database

### Steps

1. Push repository นี้ขึ้น GitHub, GitLab หรือ Bitbucket
2. ไปที่ Render dashboard
3. เลือก `New +` > `Blueprint`
4. เชื่อมต่อ repository นี้
5. ให้ Render อ่านค่าใน `render.yaml`
6. ตรวจสอบชื่อ service, region, plan, และ environment variables
7. กด deploy

### Important Blueprint Settings

Frontend service:

- Root Directory: `frontend`
- Build Command: `pnpm install --frozen-lockfile --prod=false && pnpm build`
- Start Command: `pnpm start`
- Node Version: `20` (recommended) หรือ `22`

Backend service:

- Root Directory: `backend`
- Build Command: `pnpm install --frozen-lockfile --prod=false && pnpm build`
- Pre-Deploy Command: `pnpm exec prisma migrate deploy`
- Start Command: `pnpm start`
- Node Version: `20` (recommended) หรือ `22`

### Environment Variables Used by the Blueprint

Frontend:

- `NODE_ENV=production`
- `NEXTAUTH_URL=https://sidespark-frontend.onrender.com`
- `NEXTAUTH_SECRET=<generated by Render>`
- `NEXT_PUBLIC_API_URL=https://sidespark-backend.onrender.com`

Backend:

- `NODE_ENV=production`
- `DATABASE_URL=<from Render database>`
- `FRONTEND_URL=https://sidespark-frontend.onrender.com`
- `JWT_SECRET=<generated by Render>`
- `REFRESH_TOKEN_SECRET=<generated by Render>`
- `JWT_EXPIRES_IN=1h`
- `REFRESH_TOKEN_EXPIRES_IN=7d`

### What You Must Review Before Production

- เปลี่ยน URL placeholder ใน blueprint ให้ตรงกับโดเมนจริงของ service
- ตรวจสอบว่า `FRONTEND_URL` ตรงกับ frontend domain จริง
- ตรวจสอบว่า `NEXT_PUBLIC_API_URL` ชี้ backend domain จริง
- ตรวจสอบแผนบริการของ PostgreSQL และ Web Services ให้เหมาะกับ workload

## Option B: Manual Render Setup

## 1. Create PostgreSQL Database

1. ไปที่ Render dashboard
2. เลือก `New +` > `PostgreSQL`
3. ตั้งชื่อ database เช่น `sidespark-db`
4. สร้าง database ให้เรียบร้อย
5. คัดลอก connection string ไว้ใช้เป็น `DATABASE_URL`

## 2. Create Backend Web Service

สร้าง Web Service สำหรับ backend ด้วยค่าต่อไปนี้:

- Root Directory: `backend`
- Environment: `Node`
- Build Command (recommended เมื่อ Render ไม่มี Pre-Deploy): `pnpm install --frozen-lockfile --prod=false && pnpm exec prisma migrate deploy && pnpm build`
- Start Command: `pnpm start`
- Node Version: `20` (recommended) หรือ `22`

Optional (ถ้ามี Pre-Deploy Command):

- Build Command: `pnpm install --frozen-lockfile --prod=false && pnpm build`
- Pre-Deploy Command: `pnpm exec prisma migrate deploy`

Environment Variables:

- `NODE_ENV=production`
- `DATABASE_URL=<Render Postgres connection string>`
- `FRONTEND_URL=https://<your-frontend-domain>`
- `JWT_SECRET=<random-secret>`
- `REFRESH_TOKEN_SECRET=<random-secret>`
- `JWT_EXPIRES_IN=1h`
- `REFRESH_TOKEN_EXPIRES_IN=7d`

Optional:

- `FRONTEND_URLS=https://preview-1.example.com,https://preview-2.example.com`

ใช้ `FRONTEND_URLS` เมื่อต้องเปิด CORS ให้หลาย origin

## 3. Create Frontend Web Service on Render

ถ้าจะรัน frontend บน Render ด้วย:

- Root Directory: `frontend`
- Environment: `Node`
- Build Command: `pnpm install --frozen-lockfile --prod=false && pnpm build`
- Start Command: `pnpm start`
- Node Version: `20` (recommended) หรือ `22`

Environment Variables:

- `NODE_ENV=production`
- `NEXTAUTH_URL=https://<your-frontend-domain>`
- `NEXTAUTH_SECRET=<random-secret>`
- `NEXT_PUBLIC_API_URL=https://<your-backend-domain>`

## Deploy Order

ถ้า deploy ทั้ง frontend และ backend บน Render:

1. สร้าง PostgreSQL ก่อน
2. Deploy backend ก่อน
3. รอให้ backend ใช้งานได้และ `/health` ตอบกลับ
4. Deploy frontend
5. ทดสอบ login และ API calls

## After Deploy

ตรวจสอบสิ่งต่อไปนี้:

### Backend

- เปิด `https://<backend-domain>/health` แล้วต้องได้ JSON response
- logs ต้องไม่ขึ้น error เรื่อง `FRONTEND_URL`, `JWT_SECRET`, หรือ `DATABASE_URL`
- migration ต้องรันผ่านก่อน backend start (ผ่าน Pre-Deploy หรือ Build Command)

### Frontend

- เปิดหน้าเว็บแล้วโหลดได้ปกติ
- login สำเร็จ
- เรียกข้อมูลจาก backend ได้
- session ของ NextAuth ใช้งานได้

## Troubleshooting

### Build fails because of package manager mismatch

ตรวจสอบว่า Render ใช้ `pnpm` ตามคำสั่ง build ที่กำหนดไว้ และ repository มี `pnpm-lock.yaml`

### Build fails with `EROFS: read-only file system, unlink '/usr/bin/pnpm'`

ตรวจสอบ:

- ไม่มี `corepack enable` ใน Build Command
- ใช้คำสั่ง `pnpm ...` โดยตรง
- ตั้ง Node Version เป็น `20` หรือ `22`
- redeploy ใหม่พร้อม clear build cache

### Render UI has no Pre-Deploy command

ใช้ค่าต่อไปนี้แทน:

- Build Command: `pnpm install --frozen-lockfile --prod=false && pnpm exec prisma migrate deploy && pnpm build`
- Start Command: `pnpm start`

### Backend starts but frontend calls fail

ตรวจสอบ:

- `NEXT_PUBLIC_API_URL`
- `FRONTEND_URL`
- CORS logs ใน backend

### Login fails in production

ตรวจสอบ:

- `NEXTAUTH_URL`
- `NEXTAUTH_SECRET`
- backend login endpoint
- domain ที่ตั้งใน `FRONTEND_URL`

### Prisma migration does not run

ตรวจสอบว่า backend service มี:

- `DATABASE_URL`
- คำสั่ง `pnpm exec prisma migrate deploy` อยู่ใน Pre-Deploy หรือ Build Command
