# เอกสารประกอบการทดสอบระบบส่วนหน้า (Frontend Testing Documentation)

เอกสารฉบับนี้อธิบายชุดการทดสอบ (Test Suites) ทั้งหมดที่ใช้สำหรับการตรวจสอบความถูกต้องของระบบส่วนหน้า (Frontend) ในโปรเจกต์ SideSpark

## โครงสร้างโฟลเดอร์การทดสอบ

การทดสอบของ Frontend จะถูกจัดเก็บอยู่ในโฟลเดอร์ `tests/frontend/` โดยแบ่งตามประเภทของการทดสอบดังนี้:
- `unittest/`: สำหรับการทดสอบระดับย่อย (Unit Testing) ของแต่ละส่วนประกอบ
- `integration/`: สำหรับการทดสอบการทำงานร่วมกันของหลายๆ ส่วนประกอบ
- `e2e/`: สำหรับการทดสอบการทำงานครอบคลุมทั้งระบบ (End-to-End Testing)

## การทดสอบระดับย่อย (Unit Tests)

### 1. Components (`tests/frontend/unittest/components/`)
* **`Button.test.tsx`**
  * **รายละเอียด**: ทดสอบการทำงานของปุ่ม (`Button` component)
  * **กรณีทดสอบ**: 
    * ตรวจสอบการแสดงผลผลข้อความปุ่มอย่างถูกต้อง
    * ตรวจสอบเหตุการณ์ (Event) การคลิกปุ่ม
    * ตรวจสอบสถานะการปิดใช้งาน (Disabled state)

### 2. Authorization & Authentication (`tests/frontend/unittest/auth/`)
* **`options.test.ts`**
  * **รายละเอียด**: ทดสอบการตั้งค่าและตัวเลือกสำหรับการยืนยันตัวตน (Authentication options)
  * **กรณีทดสอบ**:
    * ตรวจสอบการรับค่าคอนฟิกสำหรับ NextAuth หรือระบบ Auth 
    * ตรวจสอบตัวแปรสภาพแวดล้อมที่เกี่ยวข้อง

### 3. API Calls & Library (`tests/frontend/unittest/lib/`)
* **`api.test.ts`**
  * **รายละเอียด**: ทดสอบโมดูลที่จัดการการเรียกใช้ API หลัก
  * **กรณีทดสอบ**:
    * ยืนยันการทำงานของ instance ที่ใช้เรียก API (เช่น Axios หรือ Fetch)
    * ตรวจสอบการส่ง Headers อย่างถูกต้อง
    * ตรวจสอบจัดการข้อผิดพลาดจากการเรียก API พื้นฐาน

* **`api.auth.test.ts`**
  * **รายละเอียด**: ทดสอบฟังก์ชัน API ที่เกี่ยวข้องกับการรับรองความถูกต้อง
  * **กรณีทดสอบ**:
    * ทดสอบฟังก์ชันการเข้าสู่ระบบ (Login)
    * ทดสอบการลงทะเบียน (Register) 
    * ตรวจสอบการจัดการ Token หรือ Session เมื่อเข้าใช้งาน

## คำสั่งที่ใช้ในการรันการทดสอบ 

ที่ root directory หรือ folder `frontend/` สามารถรันคำสั่ง:
```bash
# เพื่อรันการทดสอบทั้งหมดของ frontend
pnpm test 

# เพื่อรันเฉพาะ unit testing
pnpm test:unit
```
